# Jing

> **打造你的专属 AI 伙伴** — 让虚拟角色真正"活"起来

Jing 是一个下一代智能虚拟人后端框架，基于 [pipecat](https://github.com/pipecat-ai/pipecat) 构建。不同于传统的角色扮演系统，Jing 在认知层做了深度探索 —— 让 AI 拥有持续的情绪状态、层级化的长期记忆、语义驱动的动作表现。

---

## 核心差异

Jing 解决的问题不是"如何让 AI 扮演一个角色"，而是"如何让 AI 真正像个人"。

| 能力 | Jing | 传统方案 |
|------|------|---------|
| **情绪系统** | 物理模拟引擎（惯性 + 衰减 + 边界） | 每轮重新判断情绪标签 |
| **长期记忆** | 层级金字塔结构（年→月→周→日→事件） | 向量检索或 Buffer Memory |
| **时间感知** | 时间锚检测（自动跳级查询） | 无时间维度 |
| **动作触发** | 语义驱动匹配 | 关键词或手动触发 |
| **情绪记忆** | 心跳事件（情绪状态快照） | 只记住内容，不记住心情 |
| **自主行为** | 每日生活调度（角色主动发起互动） | 纯被动响应 |
| **多平台** | QQ Bot 等 IM 渠道统一接入 | 单平台 |

---

## 核心特性

### 物理级情绪引擎

基于 PAD 三维情绪模型，情绪是连续的物理量 —— 有惯性、有衰减、有边界。开心的对话后情绪不会立刻归零，重启服务后状态从上次继续。就像真实人类。

### 层级记忆金字塔

参考人类记忆方式，多层索引、逐级衰减、适当遗忘。不只记住发生了什么，还记住当时的心情。

### 语义驱动的动作系统

不是关键词匹配，是多维度语义理解。用户说"开心地跳起来"，AI 找到真正合适的跳跃动作。支持 MMD (VMD) 动作格式。

### 每日生活系统

角色不再是"等待用户说话"。根据时间段主动发起问候、提醒、闲聊 —— 更像一个"室友"而非"客服"。

### 可插拔 Provider 层

LLM、ASR、TTS、图片生成、视频生成 —— 所有 AI 能力通过统一的 Provider 抽象接入，切换提供商只需修改配置。

## 技术栈

基于 pipecat 构建，FastAPI + WebSocket + PostgreSQL(pgvector) + Redis，支持 LiteLLM / Cerebras / Qwen ASR / CosyVoice / Cartesia / DashScope 等。

python main.py
