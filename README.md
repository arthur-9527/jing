# Jing

> **打造你的专属 AI 伙伴** — 让虚拟角色真正"活"起来

Jing 是一个下一代智能虚拟人后端框架，基于 [pipecat](https://github.com/pipecat-ai/pipecat) 构建。不同于传统的角色扮演系统，Jing 在认知层做了深度探索 —— 让 AI 拥有持续的情绪状态、层级化的长期记忆、语义驱动的动作表现。

---

## 核心差异

Jing 解决的问题不是"如何让 AI 扮演一个角色"，而是"如何让 AI 真正像个人"。

| 能力 | Jing | 传统方案 |
|------|------|---------|
| **情绪系统** | 物理模拟引擎（惯性 + 衰减 + 边界） | 每轮重新判断情绪标签 |
| **长期记忆** | 层级金字塔结构（年→月→周→日→事件） | 向量检索或 Buffer Memory |
| **时间感知** | 时间锚检测（自动跳级查询） | 无时间维度 |
| **动作触发** | 语义驱动匹配 | 关键词或手动触发 |
| **情绪记忆** | 心跳事件（情绪状态快照） | 只记住内容，不记住心情 |

---

## 核心特性

### 物理级情绪引擎

基于 PAD (Pleasure-Arousal-Dominance) 模型构建，情绪被建模为连续的物理量：

- **情绪有惯性** — 开心的对话后，情绪不会立刻归零
- **情绪有衰减** — 没有新输入时，情绪缓慢回归基线
- **情绪有边界** — 极端情绪会自然饱和，不会无限累积
- **状态持久化** — 重启服务后，情绪从上次状态继续

就像真实人类 —— 你不会因为换个话题就瞬间从暴怒变平静。

---

### 层级记忆金字塔

类似人类的记忆分层结构：

```
              年度索引（人生大事）
             ↙
          月度索引（重要经历）
         ↙
      周索引 → 日记 → 心跳事件 → 关键事件 → 对话记录
```

**时间锚检测**

用户说"去年冬天我们去了海边"，系统会自动从年度索引开始，逐级下钻找到相关记忆。这不是向量相似度能做到的 —— 向量分不清"去年"和"刚才"。

**情绪记忆**

不只记住发生了什么，还记住当时的心情（PAD 状态快照 + 强度）。下次回忆时，AI 知道"你上次说这件事时很开心"。

---

### 语义驱动的动作系统

不是预定义的关键词触发。Jing 用多维度语义匹配：

- 输入"轻轻微笑" → 自动找到最合适的微笑动作
- 结合语义理解、情绪状态、动作特征综合评分
- 支持 MMD (VMD) 动作格式导入

用户说"开心地跳起来"，AI 会找到真正合适的跳跃动作 —— 不是关键词匹配，是语义理解。

## 快速开始

```bash
# 克隆项目
git clone https://github.com/arthur-9527/jing.git
cd jing

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env 填入你的 API Keys

# 启动服务
python main.py
```

**详细部署指南**：请查看 [DEPLOY.md](./DEPLOY.md)
